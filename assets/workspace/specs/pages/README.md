# Page Specs

Create one file per page:

- `<page-slug>.md`

Each page spec should reference:

- its source
- its primary user task and CTA
- its UX success criteria and final UX review outcome
  Use `pass`, `needs-rework`, or `fail` for the review outcome.
- its header/navigation and footer decisions
- its copy discipline review and layout discipline review
- the shared theme system
- the current theme memory
- its mockup folder
- unresolved questions
- implementation decisions
- token changes
